---
name: Inspect-as-overlay (top-right or bottom-right) when on right pane
description: Replace the right-side splitter layout with a freely-resizable floating overlay anchored to either the top-right or bottom-right corner when Inspect is selected on the right pane. Adds a corner-toggle and close button on the overlay's drag bar. Also drops the Inspect-on-left option entirely.
---

# Context

Today, when the right pane is `Inspect`, the editor area is split 50/50 by a `OnePixelSplitter`. Inspect's pretty-printed JSON usually only needs a fraction of that width — so the splitter wastes screen real estate that the left pane (the actual log table) could be using.

The user wants two changes:

1. When right pane = `Inspect`, the left content fills the entire editor area, and Inspect floats as an overlay anchored in **one of two corners** of the editor — top-right or bottom-right. The user picks the corner via a small toggle on the overlay; no free dragging. The overlay is **freely resizable**, with width/height/corner persisted across sessions. The overlay's title bar also has a small **close button** that hides Inspect (equivalent to flipping the right panel to "Off" via the toolbar).
2. Remove the **Inspect-on-left** option entirely. Inspect can only be shown on the right.

# Approach

Two tightly coupled changes, both in `JsonlEditor.kt`:

1. **Drop Inspect-on-left.** Filter `Pane.INSPECT` out of the left-side toolbar group, coerce any persisted `INSPECT` value for `leftPane` (from older sessions or saved editor state) back to `FORMATTED`, and remove the now-dead cross-side fixup branches in `select()`.
2. **Inspect-as-overlay on right.** When `rightPane == INSPECT`, replace the splitter with a `JLayeredPane`-based container. Default layer holds the left component (sized to fill); a top layer holds the Inspect editor wrapped in a small chrome panel with a corner-toggle button and a SE resize grip.

A single `InspectOverlay` inner class (new, inside `JsonlEditor.kt`) owns the overlay panel and the resize/corner-toggle gestures.

## Files to modify

- `src/main/kotlin/com/olegs/jsonl/JsonlEditor.kt` — only file touched.

## Changes to `JsonlEditor.kt`

### 1. Remove Inspect-on-left

- **Toolbar group (line 208)**: change
  `Pane.entries.filter { it != Pane.NONE }.forEach { add(PickPaneAction(Side.LEFT, it)) }`
  to also filter out `Pane.INSPECT`. The left side will only show Raw/Formatted toggles.
- **Initial state (lines 122–123)**: extend the existing fallback so `leftPane` coerces both `NONE` and `INSPECT` to `FORMATTED` on load.
- **`setState()` (line 622)**: same coercion when restoring a `JsonlFileEditorState` — if `state.leftPane == INSPECT`, treat it as `FORMATTED`.
- **`select()` (lines 256–270)**: drop the now-dead `if (pane == Pane.INSPECT && rightPane != Pane.RAW)` branch in the `Side.LEFT` arm (left can no longer be Inspect). Drop the symmetric `if (pane == Pane.INSPECT && leftPane != Pane.RAW)` guard in the `Side.RIGHT` arm — with left restricted to Raw/Formatted, both are valid neighbors, so no fixup is needed.
- **`inspectVisible()` (line 333)**: simplify to `rightPane == Pane.INSPECT`.
- **`drivingEditor()` (lines 312–320)**: remove `leftPane == Pane.INSPECT` checks (none exist explicitly today, but verify the `formattedVisible`/`rawVisible` logic still resolves correctly with left ∈ {RAW, FORMATTED}).
- **`tooltipFor()` (line 684)** for `Pane.INSPECT`: change the wording from "on the $sideWord" to a fixed "on the right" since side is no longer variable.

### 2. New persistence keys (companion object near line 654)

```
const val KEY_OVERLAY_W = "com.olegs.jsonl.inspectOverlay.width"
const val KEY_OVERLAY_H = "com.olegs.jsonl.inspectOverlay.height"
const val KEY_OVERLAY_CORNER = "com.olegs.jsonl.inspectOverlay.corner"   // "TOP" or "BOTTOM"
```

Defaults when keys are missing: corner = `TOP`; width = `(0.35 * containerW).coerceIn(320, containerW - 200)`; height = `(0.60 * containerH).coerceIn(200, containerH - 32)`. Margin from the anchored corner is a fixed `8px` (no need to persist — corner-anchored).

### 3. New top-level enum

```
enum class OverlayCorner { TOP, BOTTOM }
```

### 4. `applyLayout()` (current line 277) — add overlay branch

Replace the splitter assignment with:

```
val content: JComponent = when {
    left != null && rightPane == Pane.INSPECT -> buildInspectOverlayLayer(left)
    left != null && right != null -> OnePixelSplitter(false, SPLITTER_PROPORTION_KEY, 0.5f).apply {
        firstComponent = left; secondComponent = right
    }
    left != null -> left
    right != null -> right       // unreachable post-cleanup: left is always non-null
    else -> formattedEditor.component
}
```

### 5. New `buildInspectOverlayLayer(left: JComponent): JComponent`

Returns a `JLayeredPane` with a custom `LayoutManager` whose `layoutContainer` does:

1. Size `left` to fill the full container bounds.
2. Read overlay width/height/corner, clamp width/height to fit the container, position the overlay panel against the chosen corner with an 8px inset.

Adds `left` at `JLayeredPane.DEFAULT_LAYER` and the overlay at `JLayeredPane.PALETTE_LAYER`.

The overlay panel (`InspectOverlay`) is constructed once per editor and reused — held as a lazily-initialized field on `JsonlEditor` — so toggling Inspect off/on doesn't recreate the editor view inside it.

### 6. New private inner class `InspectOverlay : JPanel`

Layout: `BorderLayout`.
- **NORTH** — `dragBar`: a thin (~22px) tinted `JPanel` containing:
  - A `JLabel("Inspect")` on the left.
  - On the right, two small icon buttons (`InplaceButton` or plain `JLabel` with `MouseAdapter`):
    1. **Corner toggle**: uses `AllIcons.Actions.MoveDown` when corner is TOP (clicking moves it down) and `AllIcons.Actions.MoveUp` when corner is BOTTOM. Click persists the new corner and triggers `revalidate()` on the layered parent.
    2. **Close**: uses `AllIcons.Actions.Close`. Click calls `select(Side.RIGHT, Pane.NONE)` — same path the toolbar's "Right panel: Off" button takes — which dismisses Inspect and re-runs `applyLayout()` to remove the overlay layer.
- **CENTER**: `inspectEditor.component`.
- **Border**: 1px line border (`JBColor.border()`) wrapping a 4px empty inner padding.

**Resize grip**: paint a 10×10 chevron in the bottom-right corner inside `paintComponent`. A single `MouseAdapter` (combined `MouseListener` + `MouseMotionListener`) installed on the overlay handles the resize:

- `mouseMoved`: if pointer is inside the SE 10×10 region, set cursor to `Cursor.SE_RESIZE_CURSOR`; otherwise default.
- `mousePressed`: if pointer is in the SE region, capture starting size + starting press point, set `resizing = true`.
- `mouseDragged` while `resizing`: compute new width/height as `(startW + dx)` / `(startH + dy)` with `coerceIn(240, parentW - 16)` and `coerceIn(140, parentH - 16)`; persist to PropertiesComponent; ask the layered parent to `revalidate()` (which re-runs `layoutContainer` and re-anchors against the active corner).
- `mouseReleased`: clear `resizing`.

The drag bar deliberately does **not** install a drag listener — corner is changed only via the toggle button.

### 7. Container resize handling

The custom `LayoutManager.layoutContainer` re-clamps overlay width/height to the parent's current bounds whenever Swing calls it (parent resize). After clamping, persist any clamp-induced size changes back to PropertiesComponent so the next session opens with valid values.

### 8. Re-use existing inspect refresh wiring

No changes to caret listeners, `refreshInspect()`, or `drivingEditor()` body. The overlay just hosts the existing `inspectEditor.component`; content updates flow through unchanged.

## Edge cases handled

- **Persisted INSPECT on left from older sessions**: coerced to FORMATTED on load (init) and on `setState`.
- **Window resized very small**: `layoutContainer` clamps overlay width/height to fit; if parent is smaller than the minimum, overlay just uses parent dimensions.
- **First-time use (no persisted values)**: defaults computed from current container size; persisted on first paint.
- **Toggling away from Inspect-on-right and back**: overlay field is reused; size and corner persist.
- **Disposing the editor**: nothing extra to release — overlay only references the existing `inspectEditor.component`, which `dispose()` already releases via `factory.releaseEditor(inspectEditor)`.

# Verification

1. Run `./gradlew runIde` to launch the sandbox IDE.
2. Open a `.jsonl` file. Check the toolbar: the **Left panel** group only shows Raw/Formatted (no Inspect button).
3. Set right panel to **Inspect** via the toolbar. Confirm the left pane fills the whole editor width, with the Inspect overlay anchored top-right.
4. Click the corner-toggle button on the overlay's drag bar: overlay snaps to the bottom-right corner; the icon flips. Click again: snaps back to top-right.
5. Click the close button on the overlay's drag bar: overlay disappears, the toolbar's right-panel state becomes Off, and the left pane keeps its full width.
6. Re-open Inspect via the toolbar: overlay returns at the last chosen size/corner.
7. Drag the bottom-right 10×10 corner: overlay resizes; cursor changes to SE_RESIZE on hover. Confirm resizing in both top-right and bottom-right modes anchors correctly to the active corner.
8. Move the caret in the left pane: overlay content updates as before.
9. Toggle right pane to **Formatted**, then back to **Inspect**: overlay re-appears with the previously chosen size and corner.
10. Resize the IDE window narrower/shorter than the overlay: overlay clamps inward and stays fully visible.
11. Close the file, restart the IDE, reopen: overlay restores last size and corner.
12. Set right pane to **Raw** or **Formatted**: confirm the splitter behavior for those modes is unchanged.
13. (Migration check) If a previous session persisted `leftPane = INSPECT`, reopen and confirm left now shows Formatted with no error.
14. Run `./gradlew test` — no test changes expected; existing tests should still pass.
